## Steps Performed

### Skill set areas reviewed

1. **Overall architecture**
   - Diagrams: https://github.com/tomtom-internal/batch-service2-infra/tree/master/docs/diagrams

2. **Quota management** (significantly different from a typical sync API)
   - https://tomtom.atlassian.net/wiki/spaces/DIRECTIONS/pages/1308262582
   - Batch Matrix 1.2 – quotas & processing speed recommendations: https://tomtom.atlassian.net/wiki/spaces/DIRECTIONS/pages/233915849/Batch+Matrix+1.2+-+quotas+processing+speed+recommendations+for+clients
   - Matrix Routing v2 – quotas & processing speed recommendations: https://tomtom.atlassian.net/wiki/spaces/DIRECTIONS/pages/287900086/Matrix+Routing+v2+-+quotas+processing+speed+recommendations+for+clients

3. **Impact assessment**
   - Use several main dashboards in [Grafana](https://grafana.com/): quota usage, status codes, executor, and health indicators.
   - [Loki](https://grafana.com/oss/loki/) – logs and error logs from classic batch / matrix / [Pulsar](https://pulsar.apache.org/).
   - Graceful degradation (health checks):
     - Classic batch & matrix may not work while the new service still does.
     - Matrix v2 async may not work while sync still works.

4. **Immediate actions**
   - Component rollout
   - Region turn-off
   - Deployment of an older version

5. **Awareness of integrations between Batch/Matrix and [Apigee](https://cloud.google.com/apigee)/[Gloo](https://docs.solo.io/gloo-edge/latest/)**
   - Relatively low risk of occurrence.
   - Unlikely to be disrupted during the night (APIM changes go through CAB, typically mid-day).
   - Still important to be aware of.

6. **PIMs**
   - Jumphost + [`kubectl`](https://kubernetes.io/docs/reference/kubectl/): to check the status of the [Kubernetes](https://kubernetes.io/) cluster and pods.

7. **P1 escalation to Microsoft**
   - Example scenario: non-functioning Azure storage with no replacement available, making it impossible to disable the download Traffic Manager.

# Fire Drill: Redis

## Scenario

[Redis](https://redis.io/) pods in the `redis` namespace on the prod [AKS](https://learn.microsoft.com/en-us/azure/aks/what-is-aks) cluster (`aks153`, `westus2`) were restarted (SIGTERM), causing leader election failures and temporary unavailability. Both `redis-0` and `redis-1` pods running [Valkey](https://valkey.io/) 7.2.11 were affected. The leader-elector sidecar on `redis-1` lost the lease due to `context deadline exceeded` errors when communicating with the [Kubernetes](https://kubernetes.io/) API server.

## Issues Encountered

- Both Redis pods (`redis-0` and `redis-1`) were shut down via SIGTERM on 2025-11-17 in the ~16:16–16:31 UTC window.
- The `leader-elector` sidecar on `redis-1` lost the Kubernetes lease because API server requests timed out (`context deadline exceeded`), preventing leader lock renewal.
- `redis-0` had 4 restarts; `redis-1` had 3 restarts during the incident window.

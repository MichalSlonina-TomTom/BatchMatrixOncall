## Issues Encountered

- [StatusCake](https://www.statuscake.com/): both participants needed email invite and account creation (tracked in SPSRE-4235).
- [AlertSite](https://smartbear.com/product/alertsite/): users needed to be added to the group (done via PR #252).
- Azure Portal prod: requires PIM just-in-time access activation before accessing.
- kubectl prod: requires PIM activation before running. PIM must be approved (not by NOC) and then activated via the Azure Portal PIM blade or the #noc-pim-request Slack workflow.

## Lessons Learned

- PIM just-in-time access for prod must be requested in advance and approved by a team member (not NOC). Use the #noc-pim-request Slack workflow.
- Several external tools (StatusCake, AlertSite, PagerDuty) require separate onboarding steps outside of standard TomTom SSO provisioning.
- Always verify current monitoring cluster name in Azure Portal before running kubectl commands — cluster names include version numbers that change over time.

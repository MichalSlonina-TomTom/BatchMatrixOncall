### Kubernetes Cluster Repair Instructions

#### Step 1: Inspect Pods

```
kubectl describe pod <pod-name>
kubectl logs <pod-name>
kubectl logs <pod-name> --previous  # if pod crashed
```

#### Step 2: Increase Number of Deployment Replicas (if applicable, for OOM error)

```
kubectl get hpa
kubectl edit hpa <hpa-name>
// not applicable for scalesets
```

#### Step 3: Increase Memory (if applicable, for OOM errors)

```
# Edit deployment
kubectl edit deployment <deployment-name>
kubectl edit scaleset <scaleset-name>

# Find resources section and update:
# resources:
#   limits:
#     memory: "4Gi"    # increase this
#   requests:
#     memory: "2Gi"    # increase this

# Verify changes
kubectl rollout status deployment <deployment-name>
kubectl rollout status scaleset <scaleset-name>
```

#### Step 4: Component Rollout (restart all replicas one by one)

```
kubectl rollout restart deployment <deployment-name>
kubectl rollout status deployment <deployment-name>

kubectl rollout restart scaleset <scaleset-name>
kubectl rollout status scaleset <scaleset-name>
```

Rollout order example (e.g. pulsar-bookie):

- pulsar-bookie-0: DELETE -> wait for new instance
- pulsar-bookie-1: DELETE -> wait for new instance
- pulsar-bookie-2: DELETE -> wait for new instance

#### Step 5: Drain Node

```
# Drain problematic node (moves all pods to other nodes)
kubectl drain aks-default-10428532-vmss000000 --ignore-daemonsets --delete-emptydir-data

# Verify
kubectl get node
kubectl get pod --all-namespaces
```

**Region turn-off rules:**

- TM -> westeurope (TURNED OFF)
- TM -> northeurope (active)
- Only one of two can be disabled

US example:
- TM -> westus (TURNED OFF)
- TM -> eastus (active)
- Only one of two can be disabled

Korea example:
- TM -> korea (active)

### "Many customers got 202 Batch & Matrix alerts"

- Reviewed PagerDuty incident: https://tomtom.pagerduty.com/incidents/Q2E9E9PJJKKOHJ
- Customer identification: same client appearing under different API keys with default quotas.

## Issues Encountered

- AlertSite false positives triggering PagerDuty alerts.
- Snoonu customer (token: af73624c-3083-4dc7-82ae-284bcb92d7bd) experiencing timeouts and HTTP 400 errors around 2026-01-07T12:00–15:30 UTC.
- "Many customers got 202" alert turned out to be the same client using different API keys with default quotas.

## Lessons Learned

- Customer identification is important: the same underlying client may appear under multiple API keys — check for shared default quotas.
- AlertSite can produce false positives; verify against other monitoring sources before escalating.
